# PRD 02.5 — Chat-Triggered Ingestion And Tailoring

> **Prerequisite:** PRDs 01 and 02 must be complete.
> **Prerequisite reading:** Read `cli.py`, `agents/chat.py`, `tui/services.py`, `ingestion/github.py`, `ingestion/resume.py` before starting.
> **Baseline test:** `python -m pytest test_smoke_formal.py -q` — all tests must pass before and after.
> **Scope:** The CLI exposes ingest-resume, ingest-github, ingest-linkedin-pdf, and tailor commands. None of these are triggerable from the TUI chat. This PRD bridges that gap by wiring the ingestion and tailoring pipelines into the chat agent's tool layer and adding argument-parsing fast-paths so they run without touching the LLM.

---

## Task Checklist

### 1 — Add ingestion service functions to `tui/services.py`
- [ ] Add `ingest_resume_file(file_path: str) -> str` — validates path exists, runs `ResumeParserAgent.parse_and_save`, returns summary string
- [ ] Add `ingest_github(username: str = "") -> str` — falls back to `GITHUB_USERNAME` from config, runs `GitHubIngestor` + `ResumeParserAgent.parse_and_save`, returns summary string
- [ ] Add `ingest_linkedin_pdf(file_path: str) -> str` — validates path, runs `LinkedInIngestor.ingest_pdf` + `ResumeParserAgent.parse_and_save`, returns summary string
- [ ] Each function returns a plain-English result string (success or error) — no exceptions escape to the caller
- [ ] LinkedIn web scraping (Playwright) is explicitly out of scope — too heavy for a chat thread

### 2 — Add tool functions to `agents/chat.py`
- [ ] Add `run_ingest_resume(file_path: str) -> str` — thin wrapper around `services.ingest_resume_file`
- [ ] Add `run_ingest_github(username: str = "") -> str` — thin wrapper around `services.ingest_github`
- [ ] Add `run_ingest_linkedin_pdf(file_path: str) -> str` — thin wrapper around `services.ingest_linkedin_pdf`
- [ ] Add `run_tailor(job_input: str) -> str` — runs the full LangGraph pipeline via `graph.pipeline.build_pipeline()`, returns a formatted summary matching the CLI output style
- [ ] Register all four in `TOOL_MAP`

### 3 — Add argument-parsing fast-paths in `_semantic_command_match`
- [ ] Add regex fast-paths (no LLM call) for these patterns, extracted before the fuzzy-match block:
  - `ingest resume <path>` → extract path, call `run_ingest_resume(path)`
  - `ingest github` → call `run_ingest_github("")`
  - `ingest github <username>` → extract username, call `run_ingest_github(username)`
  - `ingest linkedin pdf <path>` or `ingest linkedin <path>` (when path ends in .pdf) → call `run_ingest_linkedin_pdf(path)`
  - `tailor <job_input>` → extract remainder, call `run_tailor(job_input)`
- [ ] These must bypass the LLM entirely — pattern match on normalized text, extract args, return tool result

### 4 — Update SHORTCUTS and COMMAND_PHRASES
- [ ] Add to `SHORTCUTS`:
  - `"ingest github"` → `lambda: run_ingest_github("")`
- [ ] Add to `COMMAND_PHRASES["get_help_text"]`:
  - `"ingest resume"`, `"ingest github"`, `"ingest linkedin"` — so fuzzy help is suggested when user types partial ingestion commands without a path

### 5 — Add tests
- [ ] `test_chat_ingest_resume_fast_path` — monkeypatch `services.ingest_resume_file` to return a known string; assert `agent.chat("ingest resume test.md")` returns that string without calling LLM
- [ ] `test_chat_ingest_github_fast_path` — monkeypatch `services.ingest_github`; assert `agent.chat("ingest github")` returns without calling LLM
- [ ] `test_chat_tailor_fast_path` — monkeypatch `run_tailor` directly; assert `agent.chat("tailor Senior Engineer at Acme")` returns without calling LLM
- [ ] `test_ingest_resume_file_missing_path` — `services.ingest_resume_file("nonexistent.md")` returns an error string, does not raise
- [ ] All existing tests must still pass

## Constraints

1. Ingestion functions run synchronously inside the `_run_chat` background thread — this is safe and consistent with existing patterns
2. The user will see "Thinking..." during ingestion — progress UX is improved in PRD-03
3. Do not add new pip dependencies
4. Each service function must be independently callable (no TUI imports) — they are shared with PRD-03's onboarding screen
5. Error messages returned to chat must be plain English, never raw tracebacks
6. The `run_tailor` function saves `tailored_output.json` and `tailored_resume.md` to the working directory, matching CLI behavior
